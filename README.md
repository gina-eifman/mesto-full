# Mesto — социальная сеть для путешественников

> **Статус:** Учебный проект, выполненный в рамках курса «Веб-разработчик» Яндекс.Практикума.  

**Демо:** 

---

## Функциональность

- **аутентификация и авторизация** — регистрация и вход с JWT-токенами;
- **редактирование профиля** — изменение имени, описания и аватара;
- **работа с карточками**:
  - добавление новых мест (название + ссылка на изображение);
  - удаление своих карточек;
  - постановка и снятие лайков (лайки сохраняются на сервере).
- **удаление карточки** доступно только её автору;
- **адаптивная вёрстка** — корректно отображается на мобильных устройствах, планшетах и десктопах;
- **клиент-серверное взаимодействие** — все операции (получение, добавление, удаление, обновление) через REST API.

---

## Технологический стек

**Frontend:**
- React 18 (функциональные компоненты + хуки);
- React Router v6 (маршрутизация);
- CSS (адаптивная сетка, Flexbox, Grid);
- Create React App (сборка).

**Backend:**
- Node.js + Express.js;
- MongoDB + Mongoose (ODM);
- JWT + bcrypt (аутентификация и хеширование паролей);
- Celebrate / Joi (валидация запросов);
- Helmet, express-winston (безопасность и логирование).

## Запуск проекта локально

### Требования
- Node.js (v14+);
- MongoDB (локально установленная или облачный кластер Atlas).

### 1. Клонирование репозитория
```bash
git clone https://github.com/gina-eifman/mesto.git
cd mesto
```
### 2. Настройка бэкенда
```bash
cd backend
npm install
```
Создайте файл .env в папке backend со следующими переменными:
- PORT=3000;
- MONGO_URI=mongodb://localhost:27017/mestodb   # или строка подключения к Atlas;
- JWT_SECRET=ваш_секретный_ключ.

Запустите сервер:
```bash
npm run dev   # для разработки (с nodemon)
# или
npm start     # для продакшна
```
### 3. Настройка фронтенда
```bash
cd ../frontend
npm install
```
Запустите клиентскую часть:
```bash
npm start
```
Приложение будет доступно по адресу http://localhost:3001 (порт указан в package.json фронта). Бэкенд по умолчанию слушает http://localhost:3000.
Для корректной работы фронта с бэком убедитесь, что в коде фронта (например, в utils/api.js) указан правильный базовый URL бэкенда (в режиме разработки — http://localhost:3000).


## Структура проекта (кратко)

mesto/
- backend/
  - controllers/   # Обработчики запросов
  - models/        # Модели Mongoose (User, Card)
  - routes/        # Маршруты Express
  - middlewares/   # Мидлвары (авторизация, валидация, обработка ошибок)
  - errors/        # Классы ошибок
  - app.js         # Точка входа бэкенда
- frontend/
  - src/
    - components/ # React-компоненты
    - contexts/   # Контекст текущего пользователя
    - images/     # Иконки и аватар по умолчанию
    - styles/     # CSS файлы
    - utils/      # API-запросы, вспомогательные функции
    - index.js    # Точка входа фронта
  - public/
  - static/
- README.md
- README.en.md
